文章提出并实操了一套解决AI生成UI风格不稳定问题的方案：通过Claude Code搭建设计系统（Design System），将组件规范、颜色变量、间距规则等结构化信息作为上下文输入模型，让UI生成从「每次碰运气」变为「按规则稳定输出」；文章提供完整操作流程与可复用的提示词模板，演示从设计规范文档到一致性界面输出的全链路。

> 将设计系统结构化输入AI是提升UI生成一致性的方法论根本，比单纯优化提示词更能从源头解决风格漂移问题。
