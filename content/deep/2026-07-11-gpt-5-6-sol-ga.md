2026 年 7 月 9 日，OpenAI 将 GPT-5.6 系列正式全面开放（此前 6 月 26 日仅向少量伙伴受限预览），覆盖 ChatGPT、Codex、ChatGPT Work 与 API。系列分三档：旗舰 Sol、均衡的 Terra、快速经济的 Luna。相较预览阶段，此次不仅是"人人可用"，还带来了几项面向"让 AI 直接产出成品"的更新。

## 已知事实

- **三档与开放**：Sol（旗舰）、Terra（均衡、较低成本）、Luna（最快、最省）；已在 ChatGPT、Codex、ChatGPT Work 与 API 全面可用。
- **Ultra 模式与多智能体**：Ultra 模式用子智能体加速复杂任务，并为 Sol 引入更高的"最大推理强度"；同时推出 Programmatic Tool Calling 与测试阶段的多智能体功能，可并行运行多个子智能体、再合并为一个回复。
- **ChatGPT Work**：由 Codex 与 GPT-5.6 驱动，据报道可跨 Google Drive、SharePoint、Slack 等工具，在数小时内产出成品级的文档、表格、PPT 乃至网站。
- **Codex 并入桌面端**：Codex 被合并进 ChatGPT 桌面应用（macOS/Windows），可在普通对话、ChatGPT Work 与 Codex 间切换。
- **设计相关能力（官方/报道口径）**：据报道，GPT-5.6 在设计判断与前端上有提升，可自主生成完整 UI 界面、检查渲染效果后再打磨修正。
- **定价**：Sol 短上下文 API 约每百万 token 输入 5 美元、输出 30 美元。

## 横向与纵向定位

横向看，GPT-5.6 这次的重点不在单点画质或跑分，而在"agentic 产出"——让模型不只回答，而是跨应用把活干完、交出成品。ChatGPT Work 直接对着办公与交付场景，与设计工具侧"设计⇌代码打通"（如 Figma、字节 TRAE）从不同方向指向同一件事：把"从需求到可用成品"的链路交给 AI 编排。对设计工程与"用自然语言写前端"（vibe coding）而言，一个更强的编码/agentic 底座，会通过工具链间接抬高上限。

纵向看，这是 GPT-5.6 从"受限预览"到"正式开放"的一跳，也踩在模型演进从"对话/生成"向"多智能体协作、跨应用产出成品"的趋势上——推理强度可调、子智能体并行、产出物从"一段回答"变成"一份能用的文件或界面"。需要厘清的是：这些进步主要在通用推理、编码与 agentic 层面；其对纯视觉创作的影响更多是间接的（作为底座与生产工具），而"自主生成 UI"这类设计能力的实际水平，仍需以真实任务检验。

> ⚠️ 待核实："可自主生成完整 UI 界面并检查渲染后修正"为 OpenAI 与报道口径的能力宣称，实际表现需实测；"Ultra 默认 4 个 agent 并行"等具体数字未在官方核到，检索显示为"多子智能体（测试阶段）"，以官方为准；GPT-5.6 与 Claude、Gemini 等的对标跑分为报道口径，具体以官方与第三方评测为准；正式开放日期各源略有出入（GA 约 7/9），以官方为准。

> GPT-5.6 正式全面开放，并以 Ultra 子智能体、多智能体并行与 ChatGPT Work 把重心推向"跨应用产出成品"，标志通用模型从"对话/生成"向"agentic 交付"演进；对设计工程、前端与 vibe coding 方向，它作为更强的底座与生产工具会间接影响工作流，而"自主生成 UI"等设计能力的实效仍待验证。
